# Business Process Management (BPM) — Skill

Este skill aplica os fundamentos e práticas de BPM conforme o BPM CBOK 4.0 (ABPMP),
cobrindo ciclo de vida, notações, ferramentas de qualidade, KPIs e governança de processos.

---

## Como usar esta skill

Ao ser acionado, identifique o tipo de demanda do usuário e siga o fluxo correspondente:

| Demanda | O que fazer |
|---|---|
| Entender ou mapear um processo | Faça perguntas de levantamento → produza o mapa (cadeia de valor ou fluxograma) |
| Identificar problemas num processo | Use ferramentas de qualidade (Ishikawa, Pareto, 5W2H) |
| Melhorar um processo | Aplique o ciclo PDCA |
| Definir indicadores (KPIs) | Use o framework de indicadores SMART |
| Governança / controle de processos | Aplique o modelo de governança e dashboards |
| Projeto completo de BPM | Siga o Ciclo de Vida BPM (5 fases) |

Para detalhes de cada tema, leia os arquivos de referência em references/:
- references/conceitos-bpm.md — Definições, tipologias, hierarquia de processos, princípios BPM
- references/ferramentas-bpm.md — Ferramentas de mapeamento, PDCA, qualidade, 5W2H

## Fluxo de trabalho padrão

### 1. Entendimento da demanda
Antes de qualquer análise, confirme com o usuário:
- Qual processo está sendo analisado/melhorado?
- Qual é o problema ou oportunidade identificada?
- Qual é o contexto (área, empresa, tipo de organização)?
- O olhar é pontual (gestão de processos) ou estratégico (gestão por processos)?

### 2. Mapeamento AS-IS
- Identifique os fornecedores (inputs) e clientes (outputs) do processo
- Classifique o processo: Primário, de Apoio ou de Controle
- Decomponha: Macroprocesso → Processo → Subprocesso → Atividades → Tarefas
- Escolha a notação adequada: BPMN, Fluxograma, Value Stream Mapping etc.
- Identifique handoffs (rupturas entre funções) e gargalos

### 3. Análise de problemas
- Use Diagrama de Ishikawa para mapear causas-raiz
- Use Diagrama de Pareto para priorizar (regra 80/20)
- Use 5W2H para estruturar o plano de ação

### 4. Redesenho TO-BE
- Proponha melhorias com base nos problemas identificados
- Elimine handoffs desnecessários
- Automatize o que for passível de automação

### 5. Indicadores e controle
- Defina KPIs para cada tipo de processo
- Siga os critérios SMART para as metas
- Sugira estrutura de dashboard

### 6. Governança
- Defina papéis: dono do processo, gerente, arquiteto, analista/designer
- Estabeleça ciclo de revisão contínua (Kaizen)

---

## Princípios que devem nortear toda análise

1. Visão outside-in: olhar de fora para dentro, orientado a clientes
2. Processos devem estar alinhados à estratégia corporativa
3. O tripé pessoas + processos + tecnologia é indissociável
4. Medir para gerenciar: 'Não se gerencia o que não se mede' (Deming)
5. Melhoria contínua (Kaizen): todo processo pode sempre melhorar
6. Eliminar handoffs é prioridade — cada ruptura é um risco de falha
